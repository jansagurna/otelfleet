1. The agent enrolls with the control plane over OpAMP and receives its
   pipeline config remotely. Watch it appear under Fleet:

     kubectl -n {{ .Release.Namespace }} logs statefulset/{{ include "otelfleet-agent.fullname" . }} -f

2. Point your apps' OTLP exporter at the in-cluster agent Service:

     OTEL_EXPORTER_OTLP_ENDPOINT=http://{{ include "otelfleet-agent.fullname" . }}.{{ .Release.Namespace }}:{{ .Values.service.otlpHttpPort }}

   (gRPC on port {{ .Values.service.otlpGrpcPort }}.) No API key is needed on the app — the
   agent forwards under the customer identity it enrolled with.

3. Deploy another region by installing this chart again with that region's
   opamp.endpoint and bootstrap token, e.g.

     helm install {{ .Release.Name }}-us otelfleet-agent \
       --set opamp.endpoint=wss://us.otelfleet.example.com/v1/opamp \
       --set region=us --set bootstrapToken.value=otm_bt_…
{{- if .Values.opamp.insecureTLS }}

WARNING: opamp.insecureTLS is true — the bootstrap token is sent over a
plaintext ws:// connection. Use wss:// (insecureTLS=false) outside local dev.
{{- end }}
